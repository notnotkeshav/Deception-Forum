# Authentication & Authorization

## Access Levels

`users.accessLevel` is a `TINYINT(2)` in range 1–15.

| Level | Meaning |
|-------|---------|
| 1–4 | Standard users |
| 5+ | Can generate invite codes; receive real-time notifications |
| 12 | Upper bound for regular elevated users |
| 15 | Reserved — the VeinKeeper system account (unseeable, non-loginable) |

Moderators are tracked separately in the `moderators` table with a `role` of `super_admin`, `admin`, or `moderator`, and granular permissions in the `Access` table.

---

## Registration Flow

Signup requires an invite code generated by a user with `accessLevel >= 5`.

1. User visits `/` and requests an invite code (CAPTCHA required).
2. Redirect to `/signup?invite=<code>`.
3. On POST `/signup`: validate invite code, validate email/username/password (strict rules via `Validator::password()`), bcrypt hash, insert user, mark invite code used.
4. A TOTP secret is generated immediately; user is redirected to `/totp-setup`.

---

## Login Flow (Three-Step Auth)

```
POST /signin
  ├── credentials valid?
  │     └── YES → set $_SESSION['partial_auth'] (expires in 5 min)
  │                 partial_auth = { userId, expires, csrf_token, setup_required }
  │
  └── redirect to /verify-totp (or /totp-setup if first login)

POST /verify-totp
  ├── TOTP code valid (±30s window, RFC 6238)?
  │     └── YES → clear partial_auth, set full session:
  │                 $_SESSION['user']             ← full user row
  │                 $_SESSION['userId']
  │                 $_SESSION['token']            ← random bearer token
  │                 $_SESSION['token_expiration'] ← now + 24h
  │                 $_SESSION['moderator']        ← bool
  │                 $_SESSION['totp_verified']    ← true
  │                 $_SESSION['session_started']  ← now (set by index.php)
  │
  └── redirect to /threads
```

Backup codes (10× hex segments, Argon2ID-hashed) can substitute for the TOTP code.

## Session Expiry & Renewal

Session lifetime is defined once as `SESSION_LIFETIME_SECONDS = 150 * 60` in `index.php`.

On every request, `index.php` checks `time() - $_SESSION['session_started']`. If elapsed > lifetime:
1. Session is destroyed.
2. New session is started with `$_SESSION['session_expired'] = true`.
3. Redirect: `GET /verify-totp?action=renew&reason=expired&returnTo=<uri>`

After TOTP re-verification the user is returned to `returnTo`.

**Client-side** (`session-monitor.js`):
- Fetches `/session/check` on page load to get `session_started` and `session_lifetime`.
- Runs `checkSessionStatus()` every 60 seconds.
- Alerts at 30, 15, 10, and 5 **minutes remaining** (not elapsed).

## Middleware

Defined in `Backend/Middleware/Middleware.php`. Resolved by string key.

| Key | Class | Behaviour |
|-----|-------|-----------|
| `auth` | `AuthMiddleware` | Requires full session (`user` + `token` + non-expired `token_expiration`). Redirects partial-auth users to TOTP. |
| `guest` | `GuestMiddleware` | Blocks fully-authenticated users (redirects to `/threads`). Redirects partial-auth users to TOTP setup/verify. |
| `partial_auth` | `PartialAuthMiddleware` | Requires `partial_auth` session key, not yet fully authenticated. |
| `admin` | `Admin` | Requires `$_SESSION['token']` and `$_SESSION['moderator'] === true`. |
| `username_rate_limit` | `UsernameGenerationMiddleware` | 7 username-generation requests per IP per hour via file cache. |

## Password Rules (`Validator::password()`)

- Length: 25–255 characters
- At least 2 uppercase, 2 lowercase, 3 digits, 5 special characters
- No more than 2 consecutive identical characters
- No repeating substrings of length ≥ 3
- Must not contain the username or display name
- Must not contain patterns from a common-passwords list

---

## TOTP Implementation (`Backend/Utils/TOTP.php`)

RFC 6238 (TOTP) implemented from scratch:
- Secret: 32-char Base32 string
- Step: 30 seconds
- Window: ±1 step (90-second tolerance)
- Code: 6-digit, HMAC-SHA1
- QR URL: `otpauth://totp/<issuer>:<user>?secret=<s>&issuer=<i>`

Backup codes: 10 codes, each `XXXX-XXXX-XXXX-XXXX` (hex), hashed with Argon2ID. Used codes are marked in JSON stored on the user row.

## CAPTCHA (`Backend/controllers/captcha/index.php`)

| Setting | Value |
|---------|-------|
| Character set | `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (no ambiguous 0/O/1/I) |
| Length | 5 characters |
| Expiry | 12 seconds |
| Max attempts | 5 before lockout |
| Lockout duration | 15 minutes |

GET `/captcha` — generates image and stores code in `$_SESSION['captcha_code']`.  
POST `/captcha` — verifies input; checks lockout first, then attempt counter.

## Invite Codes

Generated by `GET/POST /generate_invite_code` (requires `auth`). Stored in `inviteCodes` table with `generatorId`, `used`, `usedBy`. One-time use.

The VeinKeeper account (`accessLevel=15`, non-loginable) can also seed invite codes for bootstrapping.
